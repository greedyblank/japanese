# 多端同步使用指南

让手机（iOS）和电脑同步使用同一份词条库。基于 **GitHub Gist 私有同步 + PWA**，零后端、零费用。

---

## 一、部署上线（一次性，让手机能访问）

当前仓库无 git remote。在 GitHub 建好仓库后：

```bash
git remote add origin git@github.com:<你的用户名>/japanese.git
git push -u origin main
```

然后：GitHub 仓库 **Settings → Pages → Source** 选 `main` 分支 / `root` 目录 → Save。
等几十秒，访问：

```
https://<你的用户名>.github.io/japanese/kana_etymology_map.html
```

> 手机用 **Safari** 打开该地址 → 分享按钮 →「添加到主屏幕」→ 桌面出现「仮名地図」图标，点开全屏、可离线，像原生 App。

## 二、配置同步（每台设备做一次）

1. **生成 GitHub Token**（仅需 `gist` 权限）：
   打开 <https://github.com/settings/tokens/new?scopes=gist&description=Japanese-Kana-Map> → Generate。
2. 网页右上角点 **☁️** → 粘贴 Token → 点「**✨ 自动创建 Gist**」。
   首台设备会自动建一个**私有** gist，并把 Gist ID 填入框中。
3. **其他设备**：同样打开 ☁️ → 粘贴**同一个** Token → 这次**粘贴第一台的 Gist ID**（不要点自动创建，否则会建第二个 gist）→ 保存。

> Gist ID 在第一台创建后会显示在 ☁️ 的「Gist ID」框；也可到 <https://gist.github.com> 的 Your gists 里，从 URL 末尾复制那串字符。

## 三、日常使用

| 操作 | 行为 |
|---|---|
| 加词 / 编辑 / 删除 | 约 1.5 秒后自动上传 |
| 打开 / 刷新 / 切回 App | 自动拉取合并 |
| 手动同步 | ☁️ →「🔄 立即同步」 |

多端并发**不会丢词**：每个词条有唯一 id + 时间戳，按「最新者胜」合并；删除通过墓碑跨端生效。

## 四、排错

| 现象 | 处理 |
|---|---|
| ☁️ 显示「拉取失败 HTTP 401」 | Token 失效或没勾 `gist` 权限，重新生成 |
| 「HTTP 404」 | Gist ID 填错，或 gist 被删 |
| 一端加词另一端没出现 | 点「立即同步」；确认两端用的是**同一个** Gist ID |
| 更新了代码但手机还是旧的 | Safari 清除该网站数据；或把 `sw.js` 里 `CACHE = 'kana-map-v1'` 改成 `v2` 强制刷新缓存 |

## 五、数据与隐私

- Token、Gist ID **仅存本机 localStorage**，不外发任何第三方。
- 词条数据存在你自己的**私有 gist**，GitHub 服务器可见（与你 git 仓库同等可见性）。
- 想完全离线不同步：关掉「自动同步」即可，仍可手动导入 / 导出 JSON。

## 六、AI 补全在手机端

`glm_proxy.py` 是本地 Python 代理，手机连不上。两个办法：

1. **先试直连**：多数情况 GLM 已支持浏览器 CORS 直连，设置里**不勾**「使用本地代理」即可。
2. **若直连不通**：把 `glm_proxy.py` 的转发逻辑搬到一个 Cloudflare Worker（免费额度足够），设置里填 Worker 地址。此项未在本次实现，需要时再做。
